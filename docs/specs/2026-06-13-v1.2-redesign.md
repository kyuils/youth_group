# 청소년부 출석부 v1.2 — 화면 구조 재설계 + 비주얼 리뉴얼 설계서

## 1. 배경과 목표

v1.1까지 기능(5단계 출석 상태, 역할 기반 권한, 새가족부)은 갖춰졌으나, 사용자 요구 중 다음이 미충족:

1. 주 단위 날짜 이동이 일반 날짜 입력칸(`<input type="date">`)이라 평일 선택이 가능하고 느림
2. '기록' 탭이 날짜별 **인원수만** 표시 — 해당 주의 출석자/결석자 **이름 명단**이 없음
3. 디자인이 차분한 업무 톤 — 청소년부 정체성에 맞는 밝고 활기찬 리뉴얼 요구

타 출석부 앱 UX 조사(Planning Center Check-Ins, Breeze, OneTap, 뉴젠, 선한나무 등) 결과를 반영해 화면 구조를 재설계하고 비주얼을 전면 리뉴얼한다.

**비범위(명시적 제외):** 주일만 표시되는 달력 UI — 사용자 결정으로 제외. 주 이동은 스테퍼로만 처리.

## 2. 화면 구조 (재설계)

기존 6탭(출석/기도제목/기록/통계/새가족부/관리자) → **3+α 탭**:

| 새 탭 | 통합 내용 |
|---|---|
| ✓ 출석 (홈) | 체크 + 요약 바 + 일괄 처리. 로그인 직후 자기 반 + 이번 주 제로 내비게이션 진입 |
| 📋 현황 | 기존 '기록'+'통계' 통합. 세그먼트 `주별 명단 ↔ 통계` 전환 |
| 🙏 기도 | 기존 '기도제목' 유지 + 새 테마 |
| 새가족부 / 관리자 | 권한자만 표시. 구조 유지 + 새 테마 |

구조 핵심 변화:

1. **전역 주 스테퍼 헤더** — `◀ 6월 2주 · 6/14 주일 ▶`. 모든 탭이 같은 "선택된 주일"을 공유. 주일 날짜만 ◀▶로 이동. 이번 주가 아닌 주를 보고 있으면 헤더 배경이 노란색으로 바뀌고 [이번 주로] 버튼 노출 (잘못된 주 입력 사고 방지).
2. **학생 상세 바텀시트** — 학생 이름 탭 → 시트: 큰 📞전화/💬문자 버튼(학생+부모 연락처), 기도제목, 올해 출석 이력. 행에 있던 전화/문자 칩은 상세 시트로 이동(행은 출석 체크에 집중). 단, **현황 화면의 결석자 옆에는 전화/문자 칩 직접 노출** (결석 확인 → 즉시 연락 흐름).

## 3. 홈(✓ 출석) 화면

```
◀  6월 2주 · 6/14 주일  ▶        ← 다른 주: 노란 배경 + [이번 주로]
황규일 반 · 10명
[ 3부 7 · 결석 2 · 미체크 1 ]    ← 요약 바 (sticky, 탭 시 미체크만 필터)
┃ 김단아            지난주 3부    ← 컬러바=현재 상태, 지난주 힌트
┃ (2부)(3부)(4부)(기타)(결석)     ← 5칩 1탭 확정, 행 높이 56px+
┃ 김승찬         ⚠ 2주 연속결석   ← 연속 결석 경고 배지
[ 전원 3부 출석 ▾ ]              ← 하단 고정 일괄 버튼
```

- 칩 1탭 = 즉시 저장 + 체크 애니메이션. `기타`/`결석`은 사유 바텀시트(현행 유지, 빈 값 허용).
- 요약 바: 상태별 카운트 + **미체크 N** (0이 되면 완료 확신 가능). 탭 시 미체크 학생만 필터.
- 지난주 상태 힌트, 2주 이상 연속 결석 ⚠ 배지 (심방 필요 학생 가시화).
- 일괄 버튼: 기본 라벨은 그 반 최빈 예배부(초기값 3부), ▾로 부 변경. 미체크 전원 적용 + 5초 실행취소 스낵바 (기존 applyBulk/undo 로직 재사용).
- 미체크 0 달성 시 "오늘 출석 완료! 🎉" 축하 배너.

## 4. 현황(📋) 화면

```
◀  6월 2주 · 6/14 주일  ▶        ← 공유 주 스테퍼
[ 주별 명단 | 통계 ]
✅ 출석 8
  3부 · 김단아, 박주안, …
  2부 · 이서연
  기타 · 최지호 (타교회 수련회)   ← 사유 병기
❌ 결석 2
  김승찬 (학원)      [📞][💬]    ← 결석자만 연락 칩 직접 노출
⬜ 미체크 0
```

- 사용자 요구 "해당 주 출석자/결석자 명단"의 본체. 주 스테퍼로 과거 주 확인, 이름 탭 → 출석 탭 이동해 해당 주 수정.
- `통계` 세그먼트: 기존 통계 화면을 새 테마로 이식.
- 관리자: 반 선택 드롭다운으로 전체 반 주별 명단 열람 (기존 `getAllAttendance` 활용).

## 5. 비주얼 테마 — "밝고 활기찬 청소년부"

| 요소 | 결정 |
|---|---|
| 배경/카드 | 따뜻한 크림 화이트 배경 + 순백 카드 |
| 포인트 | 코랄–오렌지 비비드 + 헤더 선라이즈 그라데이션 |
| 상태 색상 | P2 초록 / P3 파랑 / P4 보라 / ETC 주황 / ABSENT 빨강 **유지**, 한 단계 밝고 선명하게 (학습된 색 의미 보존) |
| 모서리 | radius 20px+, 부드러운 그림자 |
| 타이포 | Pretendard 유지. 학생 이름 17px bold, 본문 16px+ |
| 디테일 | 칩 체크 애니메이션, 완료 축하 배너, 이모지 빈 화면 |

접근성 원칙 (전 화면 공통): 색+텍스트 라벨 이중 부호화, 터치 타깃 44pt 이상(행 56px+), 탭 즉시 시각 피드백, 자동 저장 + "저장됨" 토스트.

## 6. 기술 계획

- **아키텍처 유지**: `web/index.html` 단일 파일 React + GitHub Pages + GAS. **GAS 코드 무수정** (필요 API 전부 존재: getRoster, getAttendance, setAttendance(+etcText), setAttendanceBatch, getPrayers, getStudentDetail, getAllAttendance).
- **API 동결**: callApi 액션·파라미터 변경 금지. 회귀 범위를 UI로 한정.
- 주 스테퍼 상태(selectedSunday)는 앱 루트에서 관리, 탭 간 공유. 기본값 = 직전(포함 오늘) 주일.
- 지난주 힌트·연속 결석 배지·최빈 예배부: 이미 로드하는 연간 출석 데이터(getAttendance year)에서 클라이언트 계산. 추가 API 호출 없음.

## 7. 에러 처리

- 저장 실패: 해당 칩에 ⟳ 재시도 표시, 탭으로 재전송. 실패 상태는 저장된 것처럼 보이지 않게 구분.
- 네트워크 끊김: 토스트 안내 (주일 아침 교회 와이파이 대비).
- 일괄 적용 실패: 부분 성공 시 실패 학생만 ⟳ 표시.
