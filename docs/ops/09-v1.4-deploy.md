# v1.4 배포 절차 (운영자용)

v1.4는 **시트 스키마 변경 + GAS 재배포**가 필요하다. 아래 순서를 지켜야 새 기능이 정상 동작한다.
이 환경에서는 라이브 시트/배포를 테스트할 수 없으므로, 배포 후 **실기기에서 직접 검증** 필요.

## 1) GAS 코드 반영
`gas/` 6개 파일(Code/Auth/Actions/Sheet/Migration/Tests)을 Apps Script 프로젝트에 복사·저장.

## 2) 마이그레이션 실행 (시트 구조 변경) — **재배포 전에 먼저**
Apps Script 편집기에서 함수 **`a_run_v14`** 실행 (권한 승인 필요).
- `새가족부` 탭 생성 + `2026 반편성`의 기존 `새가족`(P열) 인원을 새가족부로 **복사**(원본 비파괴).
- `PRAYERS` 탭에 `응답`, `응답일` 컬럼 추가.
- 실행 후 로그에 "새가족부 복사 N명 / 기도 응답 컬럼 +M" 확인.

> ⚠ 기도제목 응답 토글(#1)은 `응답`/`응답일` 컬럼이 있어야 저장된다. 마이그레이션을 먼저 돌릴 것.

## 3) 새 배포(deploy) 생성
"배포 관리 → 새 버전"으로 웹앱 재배포. `doGet`으로 `build`가 `v1.4.0-...`인지 확인.
