# v1.5 배포 절차 (운영자용)

v1.5는 **GAS 재배포**가 필요하다(시트 스키마 변경 없음). 마이그레이션은 **선택**이다.
이 환경에서는 라이브 시트/배포를 테스트할 수 없으므로, 배포 후 **실기기에서 직접 검증** 필요.

## 변경 요약 (v1.5)

- **R1** 교사가 자기 반에 일반 학생 추가 — 신규 `addStudent` 액션 + '명단' 탭의 `＋ 학생 추가`.
- **R2** 학생 정보 수정을 출석 화면 밖(신규 '명단' 탭)에서 진입. 수정 로직은 기존 `updateStudent` 재사용.
- **R3** 새가족부 중복(윤현성) — 두 갈래로 모두 커버한다(중복 원인에 따라 id 일치 여부가 달라질 수 있어서):
  - **id가 같으면**: `handleGetNewcomers` dedup이 **활성 목록에서 자동 제외**(배포 즉시, 시트 조작 불필요). 단 이건 *숨김*이라 `active`는 TRUE로 남는다 → **문자적 "삭제"(영구)** 는 `삭제` 버튼이나 아래 정리 마이그레이션으로 `active=FALSE` 처리해야 한다.
  - **id가 다르면**: dedup이 못 잡으므로 카드가 그대로 보인다 → 카드의 `삭제` 버튼으로 제거(이 버튼은 새가족부 행 자체 id로 동작해 항상 해결).
- **R4** 출석탭 필터/정렬 바를 2행(필터 / 정렬)으로 분리.
- **R5** 결석은 **1탭 즉시 저장**, 사유는 `[✎ 사유]`로 **선택 입력**(필수 아님). 일괄 결석도 사유 강제 없음.

## 1) GAS 코드 반영
`gas/` 6개 파일(Code/Auth/Actions/Sheet/Migration/Tests)을 Apps Script 프로젝트에 복사·저장.

## 2) 새가족 중복 영구 정리 — 마이그레이션 실행 권장 (id 일치 케이스)
dedup은 윤현성을 **표시상 숨길 뿐** 새가족부 행의 `active`는 TRUE로 남는다. 사용자가 요청한 것은
"삭제"이고, 만약 나중에 그의 반이 비워지면 다시 활성 새가족으로 나타날 수 있으므로, **영구 삭제를
원하면** 둘 중 하나를 한다:
- (A) 앱에서 윤현성 카드가 보이면 `삭제` 버튼 클릭, 또는
- (B) Apps Script 편집기에서 **`a_cleanup_orphan_newcomers`** 실행(권장 — id가 같아 카드가 안 보이는
  경우엔 버튼을 누를 수 없으므로 이 경로가 유일한 영구 삭제 수단).
  - 가드: `등반일`이 비어 있고 + 이미 실제 반(반≠새가족)의 학생인 새가족부 행만 `active=FALSE`로 비활성.
  - ⚠ **정상 등반자(등반일 있음)는 절대 건드리지 않는다.** 로그에 "새가족 중복 정리 N명 (이름…)" 확인.

## 3) 새 배포(deploy) 생성
"배포 관리 → 새 버전"으로 웹앱 재배포. `doGet`으로 `build`가 `v1.5.0-...`인지 확인.

## 4) 실기기 검증 체크리스트
- [ ] (R1) '명단' 탭 → `＋ 학생 추가` → 이름 등 입력·저장 → 출석/명단에 즉시 표시(본인 반)
- [ ] (R2) '명단' 탭에서 학생 `정보 · 수정` → 수정 → 저장(본인 반만). 출석 탭에서도 이름 탭으로 상세 진입은 유지
- [ ] (R3) 새가족부에 윤현성이 **활성 목록에 더는 보이지 않음**(최용수 반 명단에는 정상 존재)
- [ ] (R3) **그래도 윤현성이 보이면**(id 불일치 케이스) 그의 카드에서 `삭제` 탭 → 사라짐
- [ ] (R3) 영구 삭제 확인: 2단계 마이그레이션 실행했거나 `삭제`를 눌렀고, 새가족부 행 `active=FALSE`
- [ ] (R3) 새가족 카드 `삭제` → 확인 → 목록에서 사라짐(출석·기도 이력은 시트에 보존)
- [ ] (R4) 출석탭 필터(전체/미체크/3주결석)와 정렬(이름순/학년순/최근출석순)이 **두 줄로 구분**되어 보임
- [ ] (R5) 결석 탭 1회 → 사유 입력창 없이 즉시 결석 처리됨
- [ ] (R5) 결석 처리된 학생 카드의 `✎ 사유` → 입력창(선택) → 비우고 확인해도 저장 / 입력 시 사유 기록
- [ ] (R5) 미체크 일괄 `결석`도 사유 없이 즉시 처리
- [ ] 새가족부 교사 계정: '명단' 탭이 보이지 않음(기존대로 [새가족부, 기도제목]만)

## 롤백
- 코드: 이전 배포 버전(v1.4.0)으로 되돌리기.
- 데이터: v1.5는 스키마를 바꾸지 않는다. `a_cleanup_orphan_newcomers`를 돌렸다면, 해당 새가족부 행의
  `active`를 수동으로 다시 `TRUE`로 되돌리면 복구된다(행은 삭제하지 않으므로 비파괴적).
