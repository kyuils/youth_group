## 부장이 추가로 알아둘 점

### 사용 중 발생할 수 있는 첫 문제와 안내

| 교사가 겪을 수 있는 문제 | 대응 |
|---|---|
| "확인하지 않은 앱" 경고에서 멈춤 | "고급" → "안전하지 않음으로 이동" 클릭 — 한 번만 하면 됨 |
| 로그인 후 "접근 권한이 없어요" 화면 | TEACHERS 시트에 그 분 이메일이 등록되어 있는지 확인. 띄어쓰기·대소문자 정확? |
| 명단이 비어 있음 | `2026 반편성` 시트 P컬럼에 그 교사 이름이 매핑된 학생이 있는지 확인 |
| 출석 토글 후 1초 정도 기다리면 즉시 시트에 반영 |  |
| 시트에서 학생을 막 추가했는데 앱에 안 보임 | 5분 캐시 만료 대기 또는 부장이 Apps Script IDE에서 `invalidateCache_` 실행 |

### 신학기·새가족 추가 워크플로

1. 부장이 `2026 반편성` 시트에 새 학생 행 추가:
   - **A컬럼 (번호)**: 다음 새 번호 (예: 156) — **재사용 금지**
   - **C컬럼 (이름)**: 학생 이름
   - **P컬럼 (반)**: 담당 교사 이름 (TEACHERS 의 이름과 정확히 일치)
   - 나머지 정보 채울 수 있는 만큼
2. STUDENTS 탭은 수식으로 자동 갱신됨 (즉시)
3. 캐시는 5분 후 자동 만료 (또는 `invalidateCache_` 즉시 실행)
4. 해당 반 교사가 앱 새로고침 → 새 학생이 명단에 표시됨

### 교사 신규 가입 시

1. TEACHERS 시트에 새 행 추가:
   - email: 그 교사의 Gmail
   - 이름: 정확히 `2026 반편성` 의 P컬럼 값과 일치
   - role: `teacher` (또는 `admin` for 부장급)
   - active: `TRUE`
2. 그 교사에게 위 안내 문구 발송

### naver 계정 교사 처리 옵션

현재 박세리, 이승은, 최지혜 세 분이 naver. 옵션:
1. 그 분들이 Gmail을 새로 만들고 TEACHERS 시트의 이메일 컬럼 갱신
2. 부장이 위임 입력 (그 반 학생들을 부장이 본인 반에 임시로 등록 후 출석 — 현 구조에선 비추)
3. 향후 v1.1 에서 admin이 전체 반 열람·입력 가능하게 확장
